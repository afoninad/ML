# ML Data Analysis Project
В репозитории представлены решения задач по анализу данных с использованием методов машинного обучения и интеллектуального анализа данных. Рассматриваются подходы к поиску ассоциативных правил, кластеризации, снижению размерности, обнаружению аномалий и регрессионному моделированию.
Используемые инструменты: Python, pandas, scikit-learn, mlxtend, networkx, matplotlib, seaborn.

## Анализ транзакционных данных
В работе используется набор данных TRANSACTION.csv, содержащий идентификаторы клиентов и приобретенные товары. На первом этапе выполнен анализ уникальных значений признаков:
 - количество клиентов: 1001
 - количество товаров: 20

Далее данные были преобразованы в матрицу "клиент–товар" с бинарным кодированием покупок. На ее основе с помощью алгоритма FPGrowth найдены частые наборы товаров с порогом поддержки 5%. Дополнительно отобраны наборы фиксированной длины (4 элемента). Проведен анализ наборов, содержащих продукт peppers. Для таких наборов определен максимальный по размеру частый эпизод и его поддержка.

## Ассоциативные правила
На основе частых наборов построены ассоциативные правила с порогом достоверности 0.3. Среди правил, содержащих продукт peppers в левой части, выбрано правило с максимальным значением lift. Характеристики правила:
 - поддержка: 0.08991
 - достоверность: 0.714
 - lift: 7.53
При выполнении условия правила вероятность наступления следствия существенно выше, чем при независимом выборе товаров.

## Граф правил
Для двухэлементных правил построен ориентированный граф:
 - вершины соответствуют товарам
 - размер вершины пропорционален поддержке
 - дуги отражают правила и направлены от условия к следствию
 - веса дуг соответствуют достоверности
Рассчитаны показатели центральности (коэффициент кластеризации), определен наиболее центральный элемент, а также значение метрики для продукта peppers.

## Снижение размерности
Построены двумерные представления данных:
 - линейное — методом NMF
 - нелинейное — с использованием самоорганизующихся карт (SOM)
Нелинейное отображение позволяет лучше выявить структуру данных и разделение наблюдений.

## Кластеризация (набор baseball)
Для набора данных с характеристиками игроков выполнена стандартная предобработка:
 - заполнение пропусков (KNNImputer)
 - масштабирование признаков (MinMaxScaler)
 - кодирование категориальных переменных (OneHotEncoder)
Построена иерархическая кластеризация (метрика Манхэттена, complete linkage) и соответствующая дендрограмма. Оптимальное число кластеров выбрано на основе критерия Calinski–Harabasz (pseudo-F) как первый локальный максимум.

## Визуализация кластеров и интерпретация
Для отображения структуры данных использована SOM-проекция с окраской по номеру кластера. Дополнительно применен алгоритм K-Medoids, позволяющий определить типичных представителей кластеров.

## Дополнительная обработка признаков

Для уменьшения асимметрии распределений и влияния выбросов применено логарифмическое преобразование. После преобразования наблюдается более четкая структура кластеров и снижение их перекрытия.Также выполнен отбор признаков методом VarClus, что позволило сократить размерность без существенной потери информации.

## Обнаружение аномалий (MNIST)
Рассматривается задача отделения редкого класса (цифра 6) от основного (цифра 0). Применен метод One-Class SVM, обученный только на основной выборке. Качество модели оценивается через ROC-кривую и показатель EER, значение которого удалось снизить ниже 0.2. Проведен анализ типичных и ошибочных классификаций.

## Регрессионный анализ (donations)
Рассматривается задача прогнозирования суммы пожертвования. Данные предварительно очищены и разделены на обучающую и тестовую выборки с учетом распределения отклика.
Предобработка включает:
 - заполнение пропусков
 - масштабирование
 - кодирование категориальных признаков
 - добавление индикаторов пропусков
Отбор признаков выполнен с использованием LassoLars с подбором параметра по кросс-валидации.

## Оценка модели
Проведена оценка стабильности модели:
 - анализ CV MSE
 - бутстреппинг для оценки распределения коэффициентов
 - расчет OOB ошибки

## Нелинейное моделирование
Построена нелинейная модель с использованием полиномиальных признаков и Ridge-регрессии. Подбор гиперпараметров выполнен с помощью HalvingRandomSearchCV.
